## Section 1: What is GitHub Copilot?

GitHub Copilot is an AI pair programmer powered by OpenAI's Codex model. It integrates directly into your IDE (VS Code, JetBrains, etc.) and provides real-time code suggestions based on context.

### Key Features

- **Code Completion** - Suggests entire lines or blocks of code as you type
- **Code Generation** - Creates functions, classes, and algorithms from natural language descriptions
- **Code Explanation** - Explains what existing code does in plain language
- **Debugging Assistance** - Identifies bugs and suggests fixes
- **Documentation** - Generates docstrings, comments, and README content
- **Test Generation** - Creates unit tests for existing code

### How Copilot Works

1. You write code or a comment describing what you want
2. Copilot analyzes the context (current file, open files, language patterns)
3. It sends the context to a cloud-based AI model
4. The model generates suggestions
5. Suggestions appear as ghost text in your editor
6. You accept (Tab), reject (Esc), or modify the suggestion

## Section 2: Best Practices

### Writing Effective Copilot Prompts

- **Be specific** - "Write a function that takes a list of numbers and returns the average" is better than "write a function"
- **Provide context** - Include type hints, docstrings, and variable names that describe intent
- **Use comments** - Write a comment describing the desired behavior before the code
- **Iterate** - If the first suggestion isn't right, modify your prompt or add more context

### Limitations to Understand

- Copilot can generate incorrect or insecure code
- It may produce code that resembles training data (copyright considerations)
- Always review, test, and understand generated code
- It works best with popular languages and common patterns
- Context window is limited, so very large files may not get full context
